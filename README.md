## 📝 프로젝트 소개 (Intro)

본 프로젝트는 **'자바웹프로그래밍2'** 수업을 통해 **Spring Boot** 프레임워크를 활용하여 **나만의 포트폴리오 웹사이트와 블로그형 게시판 서비스**를 구축한 결과물입니다.
단순히 코드를 따라 치는 것이 아니라, **MVC 패턴의 구조를 이해**하고 데이터베이스와 서버 간의 통신, 그리고 사용자 인증(세션/보안)까지 백엔드 개발의 핵심 로직을 직접 구현하며 학습했습니다.

-----

## 📚 학기별 학습 로드맵 & 상세 기록록

### **1. 1주차 - 오리엔테이션**

  * **학습 내용:** 한 학기 동안 진행될 스프링 부트 프로젝트의 전체적인 로드맵을 확인하고, 웹 개발의 최신 트렌드와 백엔드 개발자의 역할에 대해 이해하는 시간을 가졌습니다.

### **2. 2주차 - 개발환경 세팅**

  * **학습 목표:** 안정적인 개발을 위한 인프라 구축 및 형상 관리 기초 다지기
  * **상세 내용:**
      * **JDK 설치 및 환경변수 설정:** 자바 개발의 기본이 되는 JDK를 설치하고 시스템 환경 변수를 잡아주었습니다.
      * **IDE 설정 (VS Code):** 'Extension Pack for Java' 등 필수 플러그인을 설치하여 개발 효율을 높였습니다.
      * **Git/GitHub 연동:** 로컬 저장소와 원격 저장소를 연결하고, `pull`, `push`, `commit` 과정을 통해 코드 버전 관리의 흐름을 익혔습니다.
      * **Hello World:** 스프링 부트 프로젝트를 생성하고 첫 컨트롤러를 만들어 브라우저에 텍스트를 띄우며 서버 구동을 확인했습니다.

### **3. 3주차 - 포트폴리오 & 프론트엔드 설계**

  * **학습 목표:** 백엔드 로직을 입힐 정적 웹 페이지(UI) 구조화
  * **상세 내용:**
      * HTML5와 CSS3를 활용하여 자기소개, 기술 스택, 프로젝트 경험을 보여줄 포트폴리오 페이지를 디자인했습니다.
      * **자바스크립트 활용:** `window.close()`, `history.back()` 등의 내장 함수를 사용하여 버튼 클릭 시 팝업 닫기나 뒤로 가기와 같은 동적인 이벤트를 직접 구현했습니다.

### **4. 4주차 - 데이터베이스 MySQL 연동 및 데이터 불러오기**

  * **학습 목표:** Spring Boot와 RDBMS(MySQL) 간의 커넥션 풀 설정
  * **상세 내용:**
      * 로컬 환경에 MySQL을 설치하고 `application.properties`에 `spring.datasource.url/username/password`를 설정하여 DB와 연결했습니다.
      * **연결 테스트:** `testdb`라는 엔드포인트를 만들어 실제 DB에 있는 데이터를 웹 브라우저로 가져오는 테스트를 수행하며, JPA가 데이터를 어떻게 매핑하는지 기초를 다졌습니다.

### **5. 5주차 - 게시판 만들기 (MVC 패턴의 이해)**

  * **학습 목표:** REST API와 View Controller의 차이점 이해
  * **상세 내용:**
      * 초기에는 `@RestController`를 사용하여 JSON 데이터를 반환하는 방식을 학습했으나, 사용자가 보는 화면(HTML)을 반환하기 위해 **`@Controller`로 리팩토링**하는 과정을 거쳤습니다.
      * 이 과정을 통해 **데이터 중심의 API**와 **사용자 중심의 View**가 어떻게 다른지 명확하게 이해하게 되었습니다.

### **6. 7주차 - 게시판 만들기 2 (JPA와 CRUD 심화)**

  * **학습 목표:** Spring Data JPA를 활용한 게시판 핵심 기능 구현
  * **상세 내용:**
      * **Entity 설계:** `Board.java` 엔티티를 생성하고 `@Entity`, `@Id`, `@GeneratedValue` 어노테이션을 사용하여 DB 테이블과 객체를 매핑했습니다.
      * **Repository 패턴:** `JpaRepository` 인터페이스를 상속받아 복잡한 SQL 쿼리 없이도 `save()`, `findAll()`, `findById()`, `deleteById()` 메서드로 CRUD를 구현했습니다.
      * **Thymeleaf 바인딩:** 컨트롤러에서 `Model`에 담아 보낸 데이터를 타임리프 문법(`${board.title}` 등)을 사용해 HTML 테이블에 동적으로 출력했습니다.
      * **성과:** 단순히 글을 쓰는 것을 넘어, 수정(`update`)과 삭제(`delete`) 로직까지 서비스 계층(`BlogService`)에 구현하여 게시판의 기본기를 완성했습니다.

### **7. 8주차 - 중간고사**

  * 전반기 학습 내용에 대한 이론 및 실습 평가를 진행했습니다.

### **8. 9주차 - 게시판 검색과 페이징 구현**

  * **학습 목표:** 대량의 데이터를 효율적으로 처리하는 방법 학습
  * **상세 내용:**
      * **검색 기능:** `findByTitleContainingIgnoreCase`와 같은 JPA 쿼리 메서드를 정의하여, 제목에 특정 키워드가 포함된 게시글을 대소문자 구분 없이 검색하도록 구현했습니다.
      * **페이징(Pagination):** `Pageable` 인터페이스와 `PageRequest.of(page, size)`를 활용하여 한 페이지에 보여줄 게시글 수를 제한하고, 하단에 페이지네이션 UI를 만들어 사용자가 쉽게 페이지를 이동할 수 있도록 만들었습니다.
      * **배운 점:** 데이터가 많아질수록 `findAll()`로 모두 가져오는 것이 서버에 큰 부하를 준다는 것을 깨달았고, 페이징 처리가 서버 성능 최적화에 필수적임을 배웠습니다.

### **9. 10주차 - 로그인과 로그아웃 구현 1 (유효성 검사)**

  * **학습 목표:** 사용자 입력 데이터 검증 및 회원가입 로직
  * **상세 내용:**
      * **DTO와 유효성 검사:** `AddMemberRequest` DTO를 만들고 `@Valid`, `@NotNull`, `@Email` 어노테이션을 사용하여 회원가입 시 잘못된 데이터가 들어오지 않도록 방어 로직을 작성했습니다.
      * **비밀번호 암호화:** 보안을 위해 `BCryptPasswordEncoder`를 사용하여 비밀번호를 평문이 아닌 암호화된 문자열로 DB에 저장했습니다.

### **10. 11주차 - 로그인과 로그아웃 2 (세션 관리)**

  * **학습 목표:** HTTP Session을 이용한 상태 유지
  * **상세 내용:**
      * **세션(Session) 처리:** `HttpSession`을 사용하여 로그인 성공 시 서버에 사용자 정보를 저장(`session.setAttribute`)하고, `JSESSIONID` 쿠키를 통해 클라이언트를 식별하는 원리를 구현했습니다.
      * **로그아웃:** `session.invalidate()`를 통해 세션을 무효화하여 로그아웃 기능을 구현했습니다.
      * **화면 분기 처리:** 타임리프의 `th:if="${session.email != null}"` 문법을 활용하여, 로그인 여부에 따라 '로그인' 버튼과 '로그아웃' 버튼이 다르게 보이도록 UI를 동적으로 제어했습니다.

### **11. 12주차 - 포트폴리오 완성 (파일 업로드 & 트러블 슈팅)**

  * **학습 목표:** 파일 처리 및 프로젝트 최종 배포 시도
  * **상세 내용:**
      * **파일 업로드:** `MultipartFile` 인터페이스를 사용하여 사용자가 프로필 사진이나 메일 내용을 업로드하면 서버의 특정 경로(Static 폴더 등)에 저장하는 기능을 구현했습니다. 메일 전송 폼을 통해 내용을 텍스트 파일로 저장하는 로직도 추가했습니다.
      * **트러블 슈팅 (웹 호스팅):** \* **문제:** 프로젝트 완성 후 GitHub Pages를 통해 배포를 시도했으나, CSS가 깨지고 403 오류가 발생했습니다.
          * **원인 분석:** GitHub Pages는 **정적(Static) 파일**만 호스팅할 수 있는데, 제 프로젝트는 Spring Boot 기반의 **동적(Dynamic) 서버 사이드 렌더링** 방식이라 서버(Tomcat)가 필요했기 때문입니다. 또한 Thymeleaf 문법은 서버가 해석해야 하는데 정적 호스팅에서는 이것이 불가능했습니다.
          * **해결 시도:** `index.html`의 타임리프 문법을 제거하고 순수 HTML로 변경하여 정적 페이지로 업로드하는 방법을 시도해 보았습니다. 추후 AWS나 클라우드 서비스를 통해 실제 Spring 서버를 배포해볼 계획입니다.

## 💻 기술 스택 (Tech Stack)

| 분류 | 기술 | 설명 및 활용 |
| :--- | :--- | :--- |
| **Language** | **Java 17** | 프로젝트의 핵심 언어로 사용 |
| **Framework** | **Spring Boot 3.x** | 내장 톰캣과 자동 설정을 통한 빠른 웹 애플리케이션 구축 |
| **Database** | **MySQL** | 회원 정보 및 게시글 데이터를 저장하는 관계형 데이터베이스 |
| **ORM** | **Spring Data JPA** | SQL 중심 개발에서 객체 중심 개발로 전환, CRUD 생산성 향상 |
| **Template** | **Thymeleaf** | 컨트롤러에서 전달받은 데이터를 HTML에 렌더링하는 템플릿 엔진 |
| **Frontend** | **HTML/CSS/JS** | 사용자 인터페이스 구현 및 클라이언트 측 이벤트 처리 |
| **Tool** | **VS Code / Git** | 코드 편집 및 버전 관리 |

특히 `Controller`, `Service`, `Repository`로 이어지는 계층형 아키텍처(Layered Architecture)를 직접 설계하면서 **코드의 역할 분담이 왜 중요한지** 몸소 느꼈습니다. 1학기때 했던 자바웹 수업은 html에서 대부분 이뤄지고 백엔드 부분은 깊게 들어가지 못했는데 2학기때에는
백엔드 과정을 더 배우면서 왜 어렵다고 하는지 느꼈습니다. 하지만 그만큼 더 노력하고 공부해야겠다고 느낀 계기가 된 수업이라
방학동안 더욱 열심히 공부하겠습니다.

마지막에 웹 호스팅 과정에서 정적/동적 웹 사이트의 차이로 인해 배포를 하지 못해 아쉬웠지만, 이 실패를 통해 **서버의 필요성과 배포 환경의 차이**를 확실하게 공부하는 계기가 되었습니다. 앞으로 AWS와 같은 클라우드 환경에서 실제 서비스를 배포해보는 것을 다음 목표로 삼아 다음년도 공과대학 학생회를 하면서 게시판이나 소통창구를 실제로 구현해 볼 수 있도록 정진하겠습니다. 1년동안 감사했습니다.
